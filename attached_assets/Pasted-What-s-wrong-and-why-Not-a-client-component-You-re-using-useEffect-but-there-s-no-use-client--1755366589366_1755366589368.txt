What‚Äôs wrong (and why)

Not a client component

You‚Äôre using useEffect, but there‚Äôs no "use client" directive. In App Router, the file is server by default ‚Üí hooks won‚Äôt run.

Class name mismatches with /static/app.js

The script looks for .tab, .card, .badge, .muted, and .grid.

Your component only defines .travel-tab, .travel-card, .travel-badge, .travel-muted, .travel-grid.

Result: the script can‚Äôt find tab buttons; cards it creates (<article class="card">) are unstyled.

Script may be injected twice in Strict Mode

React Strict Mode mounts ‚Üí unmounts ‚Üí mounts again in dev. You append the script each mount. You remove it on unmount, but during quick remounts you can still end up racing two loads.

Script path must actually exist

In Next.js, static assets should be under /public. Ensure the file is at public/static/app.js so /static/app.js resolves.

(Minor) Using defaultValue while JS mutates .value

It‚Äôs fine (uncontrolled inputs) ‚Äî just be aware React won‚Äôt manage those values. This is intended since the external script sets values.

Quick fixes (minimal, safe)

Paste this updated component. It adds "use client", guards against duplicate script injection, and aliases the legacy classes so your styles apply to elements the script creates.
